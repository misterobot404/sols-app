Три вида доступа к тесту:

1. Через доменный аккаунт (преподаватель/студент).
Необходим для проверки знаний студента в рамках обучения.
Авторизация сейчас: Через собственную страницу входа с помощью данных (каких?) доменной учётки.
Авторизация в будущем: Вход происходит через единую страницу доменной авторизации и является сквозной для всех сервисов KNASTU.
Тест прикрепляется преподавателем на группу, при входе студента на сервис, получаем его учебную группу, а уже по ней - все текущие тесты.

2. Через локальный аккаунт.
Необходим для прохождения олимпиад и проверки знаний на разные темы.
Участие может принять любой желающий после создания локального аккаунта в системе.

3. Абитуриент.
Необходим для прохождения вступительного тестирования абитуриентами.
Кнопка для доступа к этой странице будет появляться только в сети университета (или публичный доступ?) на странице входа в SOLS.
Информацию о группах и абитуриентах(какие данные кроме ФИО?, что нужно сохранять после тестирования?) мы получаем из системы абитуриента.
Преподаватель при создании теста может выбрать в настройках доступа "для абитуриентов" и сгенерировать пароль (общий, личный?).
После этого тест будет отображаться на странице абитуриента где для начала теста будет необходимо ввести ФИО(или выбрать из списка?) и пароль.
